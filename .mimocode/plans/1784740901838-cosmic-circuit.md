# Rules System + React Monaco Editor (New Tab)

## Architecture Overview

Build a React app (Vite + React + Monaco Editor) that compiles into the Chrome extension. The management UI opens as a **full-page editor in a new Chrome tab** — not inside the side panel. The side panel chat is also available in that tab. The AI agent edits skills/memories/rules via existing tool calls, not browser automation.

**Flow:**
1. User clicks "Edit" button in side panel header
2. Extension opens `chrome.tabs.create({ url: chrome.runtime.getURL("editor.html") })` 
3. React app loads with list + Monaco editor
4. Side panel is automatically available in that tab (Chrome side panel is per-window)
5. AI agent can edit items via `skills`/`memories`/`rules` tool calls from the chat

## Step 1: Rules Tool — Background

**File: `background.js`** (after `handleSkills`, before `// --- Page Tools ---`)

Add `handleRules(args)` — mirror of `handleMemories()`:
- Storage key: `"agent_rules"`
- Data shape: `{ rules: [{ id, title, content, created, updated }] }`
- ID format: `rule_${Date.now()}_${Math.random().toString(36).slice(2,8)}`
- Actions: `list`, `read`, `write`, `delete`

Add dispatch in `handleExecuteTool()` (after `skills` check, before `!tabId` check):
```js
if (tool === "rules") {
  return await handleRules(args);
}
```

## Step 5: Chrome Storage Integration

### `hooks/useStorage.js`
The React app communicates with the background script via `chrome.runtime.sendMessage`:

```js
export function useStorage(toolName) {
  const [items, setItems] = useState([]);
  const [loading, setLoading] = useState(true);

  const loadList = async () => {
    const result = await chrome.runtime.sendMessage({
      type: "executeTool",
      tool: toolName,
      args: { action: "list" }
    });
    if (result.ok) setItems(result.data[toolName] || []);
    setLoading(false);
  };

  const readItem = async (id) => {
    const result = await chrome.runtime.sendMessage({
      type: "executeTool",
      tool: toolName,
      args: { action: "read", id }
    });
    return result.ok ? result.data : null;
  };

  const writeItem = async (args) => {
    const result = await chrome.runtime.sendMessage({
      type: "executeTool",
      tool: toolName,
      args: { action: "write", ...args }
    });
    if (result.ok) await loadList();
    return result;
  };

  const deleteItem = async (id) => {
    const result = await chrome.runtime.sendMessage({
      type: "executeTool",
      tool: toolName,
      args: { action: "delete", id }
    });
    if (result.ok) await loadList();
    return result;
  };

  return { items, loading, loadList, readItem, writeItem, deleteItem };
}
```

## Step 6: Editor Page in Chrome Extension

### `manifest.json` updates:

Add `editor.html` to web-accessible resources and update CSP:

```json
{
  "web_accessible_resources": [{
    "resources": ["editor/**"],
    "matches": ["<all_urls>"]
  }],
  "content_security_policy": {
    "extension_pages": "script-src 'self' 'wasm-unsafe-eval'; object-src 'self'"
  }
}
```

### New file: `editor.html` (at extension root)
A thin HTML shell that loads the React build:
```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>Agent Editor</title>
  <link rel="stylesheet" href="editor/assets/index.css" />
</head>
<body>
  <div id="root"></div>
  <script type="module" src="editor/assets/index.js"></script>
</body>
</html>
```

### `background.js` — add editor tab opener:
```js
if (message.type === "openEditor") {
  await chrome.tabs.create({
    url: chrome.runtime.getURL("editor.html")
  });
  sendResponse({ ok: true });
  return;
}
```

### Side panel — add "Edit" button:
In `sidepanel.html`, add to `.header-actions`:
```html
<button id="editBtn" class="btn small" type="button">Edit</button>
```

In `sidepanel.js`, add listener:
```js
dom.editBtn.addEventListener("click", () => {
  chrome.runtime.sendMessage({ type: "openEditor" });
});
```
